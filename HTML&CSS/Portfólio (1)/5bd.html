A matéria de Banco de Dados abordou conceitos fundamentais e práticos ao longo do semestre.
Iniciamos com a estrutura de dados, onde aprendemos sobre diferentes tipos de dados e como eles são organizados e manipulados dentro de um banco de dados relacional. Exploramos modelagem de dados, incluindo diagramas ER (Entidade-Relacionamento), para visualizar a estrutura e as relações entre dados.

Estudamos os principais conceitos de banco de dados, como normalização, chaves primárias e estrangeiras, índices, transações e integridade referencial. Esses conceitos são cruciais para a criação de bancos de dados eficientes, consistentes e seguros.

Utilizamos o PgAdmin, uma ferramenta de gerenciamento de bancos de dados PostgreSQL, para realizar tarefas práticas. Aprendemos a criar e gerenciar bancos de dados, tabelas, e relacionamentos. Executamos consultas SQL para inserir, atualizar, deletar e recuperar dados, bem como realizar junções complexas e agregações.

PROJETOS

Projeto Sistema Educacional:

No projeto realizado no PgAdmin, fomos contratados para projetar o banco de dados relacional de um sistema educacional para uma universidade.
O objetivo era criar um sistema robusto capaz de coletar e gerenciar informações detalhadas de alunos, cursos, disciplinas e departamentos.

Diagrama Entidade-Relacionamento (DER):
Foi criado um Diagrama Entidade-Relacionamento (DER) para visualizar e validar a estrutura do banco de dados com o cliente.
Este DER detalha as relações entre alunos, cursos, disciplinas e departamentos.

Consultas e Filtragens:
Para atender às necessidades dos colaboradores da secretaria da universidade, desenvolvemos várias consultas SQL para filtrar e recuperar dados específicos do sistema.

... Implementação SQL:
Para a implementação do banco de dados, criamos um arquivo .sql contendo todas as queries organizadas da seguinte forma:

▣ DDL (Data Definition Language): Comandos para criar as tabelas e definir suas relações.

▣ DML (Data Manipulation Language): Comandos para inserir dados iniciais nas tabelas.

▣ DQL (Data Query Language): Consultas para recuperar dados com base nos requisitos.

Este projeto exemplifica a criação de um banco de dados relacional para um sistema educacional, cobrindo desde o design do DER até a implementação de consultas SQL, usando PgAdmin para gestão e manipulação dos dados.

CONTEÚDOS

SQL

SQL, ou Structured Query Language, é uma linguagem de programação projetada para gerenciar bancos de dados relacionais. Ela oferece um conjunto de comandos que permitem criar, modificar e consultar dados armazenados em tabelas.

Com SQL, é possível definir a estrutura de um banco de dados (DDL), inserir, atualizar e excluir registros (DML), e realizar consultas complexas para recuperar informações específicas (DQL). SQL é amplamente utilizado em aplicações de software para manipulação de grandes volumes de dados de maneira eficiente e segura, sendo fundamental para desenvolvedores, administradores de banco de dados e analistas de dados em diversos contextos industriais e acadêmicos. Além disso, também permite realizar operações de agrupamento, ordenação e filtragem de dados.

DADOS

Estruturas de dados são modelos organizacionais utilizados para armazenar e organizar dados de forma eficiente dentro de um computador.

Elas permitem gerenciar e manipular informações de maneira estruturada, facilitando operações como inserção, busca, remoção e ordenação. Exemplos comuns incluem listas, pilhas, filas, árvores e grafos, cada uma otimizada para diferentes tipos de operações e necessidades de armazenamento de dados. O estudo de estruturas de dados é essencial para desenvolver algoritmos eficientes e sistemas robustos em programação e ciência da computação.

NOSQL

NoSQL, ou Not Only SQL, é um termo que se refere a uma variedade de sistemas de gerenciamento de banco de dados que diferem dos tradicionais bancos de dados relacionais (SQL). Entre esses, destaca-se o MongoDB, conhecido por sua flexibilidade e escalabilidade, sendo ideal para dados não estruturados.

NoSQL é projetado para lidar com grandes volumes de dados não estruturados ou semiestruturados de forma mais flexível e escalável do que os bancos de dados relacionais. Esses sistemas são frequentemente utilizados em ambientes de Big Data e aplicações web modernas, oferecendo modelos de dados mais dinâmicos como documentos, grafos, famílias de colunas e chave-valor. Um exemplo popular é o MongoDB, que é valorizado por sua capacidade de suportar velocidades de leitura e escrita elevadas.
